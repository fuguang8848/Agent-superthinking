## 6. 建议

### 6.1 短期建议 (当前迭代)

1. **完善 CLI 错误消息**: 将无效 scenario 的错误信息从 "Invalid scenario" 改为列出有效选项
2. **添加真实 LLM 测试**: 在 CI/CD 中添加使用真实 API 的冒烟测试
3. **性能基准可视化**: 集成 pytest-benchmark HTML 报告

### 6.2 中期建议 (下一迭代)

1. **端到端性能监控**: 添加 APM 集成追踪真实用户场景
2. **安全渗透测试**: 引入专业安全团队进行红队测试
3. **跨平台真机测试**: 在 Linux/macOS/Windows 上运行完整测试套件

### 6.3 长期建议 (路线图)

1. **AI 对抗性测试**: 引入专门的幻觉检测和对抗性输入测试
2. **自动回归测试**: 基于历史 bug 自动生成回归测试用例
3. **混沌工程**: 模拟专家崩溃、网络中断等故障场景
