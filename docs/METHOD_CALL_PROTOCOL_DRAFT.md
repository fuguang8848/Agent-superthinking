# 方法论调用协议草案 (METHOD_CALL_PROTOCOL_DRAFT)

> 本文档定义专家在发言中如何声明「我用 X 方法论检验」的方法论调用协议，为 V6 动态专家池和方法论集成提供规范基础。

---

## 一、协议概述

### 1.1 协议目的

方法论调用协议旨在为多专家辩论系统提供统一的方法论声明和调用规范，使得：
1. 专家能够清晰地声明其使用的方法论
2. 系统能够正确路由和整合不同方法论的分析
3. 用户能够理解专家的分析依据
4. 方法论调用可追溯、可验证

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| **显式声明** | 每次方法论调用必须显式声明 |
| **结构化输出** | 方法论分析必须遵循固定结构 |
| **可验证性** | 方法论结论必须可追溯到其依据 |
| **互操作性** | 不同方法论可组合使用 |
| **最小侵入** | 不影响专家的自然发言风格 |

## 二、声明格式规范

### 2.1 基础声明格式

专家在发言中使用方法论时，必须遵循以下声明格式：

```
[方法论调用: <方法论名称>] <分析内容> [/方法论调用]
```

**示例**：
```
[方法论调用: 伦理学]
从义务论角度看，这个决策违背了"永远将人性作为目的而非手段"的绝对命令。
具体而言，将员工数据出售给广告公司是将人仅仅作为获利工具的行为。
[/方法论调用]
```

### 2.2 扩展声明格式

对于需要多层次分析的场景，使用扩展格式：

```
[方法论调用: <方法论名称>]
**检验对象**: <待检验的论点>
**方法论框架**: <使用的具体框架>
**分析过程**: <分析步骤>
**检验结论**: <结论>
[/方法论调用]
```

**示例**：
```
[方法论调用: 分析哲学]
**检验对象**: "人工智能最终会超越人类智能"
**方法论框架**: 概念澄清 - 家族相似性原理
**分析过程**: 
1. "智能"概念无固定本质，只有重叠交叉的相似性网络
2. "超越"可指不同维度（计算速度、创造力、情感智能等）
3. 各维度上的"超越"标准不一致
**检验结论**: 该命题存在概念模糊，"智能"和"超越"都需要明确定义才能判断真伪。
[/方法论调用]
```

### 2.3 组合声明格式

当专家同时使用多个方法论时：

```
[方法论组合: 伦理学 + 法学]
**检验对象**: 政府是否应该强制接种疫苗
**伦理学视角**: 从功利主义角度，强制接种的净收益（减少疾病传播）大于成本
**法学视角**: 从自然法角度，强制侵犯了身体自主权这一基本人权
**综合结论**: 两种方法论给出不同结论，需要在具体情境中权衡
[/方法论组合]
```

## 三、方法论调用类型

### 3.1 主动调用

专家主动选择方法论进行检验：

```
[方法论调用: 传播学]
作为传播学专家，我用议程设置理论检验这个媒体策略。
议程设置理论指出：媒体不能决定人们怎么想，但能决定人们想什么。
应用到这个案例：当前的媒体策略试图将公众注意力从X转移到Y。
但由于X议题与公众切身利益相关，议程设置效果可能有限。
[/方法论调用]
```

### 3.2 响应调用

专家应主持人或用户请求进行方法论检验：

```
[响应调用: 主持人请求]
[方法论调用: 经济学]
根据主持人的请求，我用博弈论检验这个市场策略。
纳什均衡分析表明：在当前市场结构下，价格战是唯一稳定策略。
这解释了为什么各企业即使知道价格战会导致集体损失，仍无法避免。
[/方法论调用]
```

### 3.3 验证调用

专家验证其他专家的方法论结论：

```
[方法论验证: 伦理学 → 政治哲学]
我检验一下[专家A]从伦理学角度的结论。
[专家A]认为该决策符合功利主义原则。
但从政治哲学的公共理性框架检验：作为公共政策，决策依据的理由必须是所有公民都能接受的。
功利主义计算可能不符合公共理性要求，因此[专家A]的结论需要修正。
[/方法论验证]
```

## 五、方法论调用流程

### 5.1 标准调用流程

```
1. 识别需求
   ↓
2. 选择方法论
   ↓
3. 声明调用
   ↓
4. 执行分析
   ↓
5. 输出结论
   ↓
6. 结束声明
```

### 5.2 流程详解

**1. 识别需求**
- 识别问题是否需要方法论检验
- 判断哪种方法论最适用

**2. 选择方法论**
- 参考 METHODS_INDEX.md 选择方法论
- 必要时组合多种方法论

**3. 声明调用**
```
[方法论调用: <方法论名称>]
```

**4. 执行分析**
- 按方法论框架进行分析
- 保持分析的逻辑性和完整性

**5. 输出结论**
- 给出方法论视角的结论
- 说明结论的依据

**6. 结束声明**
```
[/方法论调用]
```

### 5.3 调用决策树

```
问题分析
    ↓
是否需要方法论检验？
    ↓ 是
该问题属于哪类？
    ↓
┌──────────┬──────────┬──────────┬──────────┐
↓          ↓          ↓          ↓          ↓
决策类   理解类   创意类   综合类
    ↓          ↓          ↓          ↓

伦理学    社会学    叙事学
    ↓          ↓          ↓          ↓
    └──────────┴──────────┴──────────┘
                    ↓
            声明并执行调用
```

## 六、方法论输出结构

### 6.1 标准输出模板

```
[方法论调用: <方法论名称>]

## <方法论名称> 分析

**检验对象**: <待检验的论点>

**方法论框架**: <使用的具体理论/模型>

**分析过程**:
1. <步骤1>
2. <步骤2>
3. <步骤3>

**方法论结论**: <结论>

**置信度**: <0.0-1.0>

**适用边界**: <方法论的适用范围的说明>

[/方法论调用]
```

### 6.2 简化输出模板

对于简短分析：

```
[方法论调用: <方法论名称>]
<分析内容>
**结论**: <一句话结论>
[/方法论调用]
```

### 6.3 输出示例

**完整版**：
```
[方法论调用: 伦理学]

## 伦理学分析

**检验对象**: 公司是否应该将用户数据商业化

**方法论框架**: 康德义务论 - 人性公式

**分析过程**:
1. 将用户数据商业化 = 将用户（及其行为数据）作为商业手段
2. 用户是否知情并同意其数据被用于商业目的？
3. 若用户不知情，则违背"将人性作为目的"原则

**方法论结论**: 在当前情况下，该决策违背义务论原则。

**置信度**: 0.85

**适用边界**: 本结论基于义务论框架，功利主义可能给出不同结论。

[/方法论调用]
```

## 七、方法论兼容性矩阵

### 7.1 方法论组合推荐

| 方法论对 | 组合效果 | 适用场景 |
|----------|----------|----------|
| 伦理学 + 法学 | 道德-法律双审视 | 政策评估 |
| 经济学 + 博弈论 | 理性-策略双框架 | 市场分析 |
| 社会学 + 传播学 | 结构-流动双视角 | 舆论分析 |
| 心理学 + 现象学 | 行为-体验双层次 | 用户研究 |
| 哲学 + 科学 | 思辨-实证双路径 | 技术伦理 |
| 管理学 + 控制论 | 战略-系统双维度 | 组织设计 |

### 7.2 方法论冲突检测

| 方法论对 | 潜在冲突 | 解决方案 |
|----------|----------|----------|
| 功利主义 vs 义务论 | 结果 vs 原则 | 分别声明，由用户权衡 |
| 理性主义 vs 经验主义 | 先验 vs 后验 | 组合分析 |
| 个人主义 vs 集体主义 | 原子 vs 结构 | 分层分析 |

## 八、协议扩展机制

### 8.1 自定义方法论声明

专家可以声明使用自定义方法论（项目内未收录）：

```
[方法论调用: 自定义: <方法论名称>]
**方法论来源**: <来源说明>
**核心原理**: <简要说明>
**分析**: <分析内容>
[/方法论调用]
```

### 8.2 方法论继承

当分析方法继承自某一传统时：

```
[方法论调用: 伦理学 → 亚里士多德]
从亚里士多德美德伦理学角度...
[/方法论调用]
```

### 8.3 方法论版本

当同一方法论有不同版本时：

```
[方法论调用: 伦理学 v2 → 康德义务论]
使用康德义务论第二版，增加对积极自由的考量...
[/方法论调用]
```

## 九、实施建议

### 9.1 协议采用策略

1. **试点阶段**：选择一个复杂场景试点协议
2. **逐步推广**：逐步推广到所有方法论调用
3. **工具支持**：开发协议解析和验证工具
4. **反馈迭代**：根据使用反馈优化协议

### 9.2 培训建议

1. **协议培训**：让专家熟悉协议格式
2. **示例展示**：提供各方法论的标准示例
3. **工具辅助**：提供声明模板和自动补全

### 9.3 质量控制

1. **格式验证**：自动检查声明格式
2. **内容审核**：审核方法论分析的准确性
3. **效果评估**：评估协议对辩论质量的影响
