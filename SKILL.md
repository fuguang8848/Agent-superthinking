# ⚔️ 超思考 · 双轨专家系统 v3

> 不同领域的思想家看待同一个问题，结论往往不同——这种差异本身就是洞察的来源。

## 核心架构：双轨制

```
问题输入
    ↓
┌─────────────────────────────────────────┐
│  路由层：判断涉及哪些专家团            │
│  → 展示路由结果 + 询问用户选择轨别     │
└─────────────────────────────────────────┘
    ↓
用户选择轨别：
┌─────────────────────────────────────────┐
│  [1] 人物型（推荐）                    │
│      苏格拉底/马斯克/尼采/芒格...       │
│                                         │
│  [2] 方法论型                          │
│      复杂性科学/博弈论/贝叶斯/系统论...  │
│                                         │
│  [3] 双轨组合（最全面）                │
│      人物 + 方法论一起分析              │
│                                         │
│  [4] 自定义                            │
│      自己选具体专家/框架                │
└─────────────────────────────────────────┘
    ↓
用户选择粒度：
├─ [A] 完整（所有涉及团）
├─ [B] 只要指定团
└─ [C] 指定具体专家/框架
    ↓
被选中的专家/框架并行分析
    ↓
冲突检测 + 共识提炼
    ↓
结构化综合报告
```

**关键改进：**
- 双轨制：人物型 + 方法论型分开
- 用户决定用哪轨（单轨或双轨组合）
- 架构更清晰：人物=价值观/决策，方法论=分析工具

### Step 2: 路由结果展示

**输出格式：**

```
┌─────────────────────────────────────────────────┐
│  问题涉及：                                      │
│                                                 │
│  人物型：哲学团（苏格拉底、柏拉图、尼采...）    │
│          科学团（爱因斯坦、图灵...）             │
│                                                 │
│  方法论型：复杂性科学（涌现、自组织）           │
│            博弈论（纳什均衡、囚徒困境）         │
│                                                 │
│  请选择分析轨别：                                │
│  [1] 只要人物型（推荐）                         │
│  [2] 只要方法论型                               │
│  [3] 双轨组合（最全面）                         │
│  [4] 自定义                                    │
└─────────────────────────────────────────────────┘
```

### Step 3: 用户选择轨别

| 选项 | 说明 | 适用场景 |
|------|------|---------|
| [1] 人物型 | 历史人物视角 | 价值观判断、决策启发、人性洞察 |
| [2] 方法论型 | 学术框架视角 | 分析工具、量化方法、系统建模 |
| [3] 双轨组合 | 人物 + 方法论 | 需要深度+广度的复杂问题 |
| [4] 自定义 | 自己选 | 有明确偏好的用户 |

### Step 4: 用户选择粒度

| 选项 | 说明 | 上下文占用 |
|------|------|-----------|
| [A] 完整 | 所有涉及团/框架全开 | 最高 |
| [B] 指定团 | 用户选1-2个团 | 中等 |
| [C] 指定专家/框架 | 用户自己选 | 最低 |

### Step 5: 并行分析

被选中的专家/框架**并行**分析问题。

### Step 6: 冲突检测

- 人物间分歧 → 价值观张力
- 方法论间冲突 → 模型冲突
- 人物 vs 方法论 → 理论与实践张力

```markdown
- 激活方法论：[列表]

### 核心发现

#### 方法论视角

**理论与实践张力**：（人物 vs 方法论）

### 最终建议
[综合所有视角的最终结论]
```

## 双轨组合示例

### 示例 1：AI会不会取代人类？

```
用户选择：[3] 双轨组合 + [A] 完整

人物型激活：哲学团 + 科学团 + 心理/社会团
方法论型激活：复杂性科学 + 演化学 + 博弈论

→ 哲学提供存在主义视角
→ 科学提供技术可能性视角
→ 心理/社会提供人类行为视角
→ 复杂性科学分析涌现性质
→ 演化学分析适应与淘汰
→ 博弈论分析人机策略互动
```

### 示例 2：要不要创业？

```
用户选择：[1] 人物型 + [B] 指定团

只激活：战略决策团（马斯克、芒格、纳瓦尔、乔布斯）

→ 马斯克：第一性原理分析
→ 芒格：逆向思考 + 多元思维
→ 纳瓦尔：杠杆思维
→ 乔布斯：聚焦判断
```

### 示例 3：这个系统为什么崩溃？

```
用户选择：[2] 方法论型 + [B] 指定框架

只激活：系统论 + 复杂性科学 + 网络科学

→ 系统论：反馈循环断裂点
→ 复杂性科学：涌现失效分析
→ 网络科学：结构脆弱性
```

## 与旧版区别

| 维度 | v1/v2 | v3 |
|------|-------|-----|
| 专家类型 | 混在一起 | 双轨分开 |
| 路由 | 判断专家团 | 判断轨别（人物/方法论/双轨） |
| 组合 | 专家组合 | 人物×方法论交叉组合 |
| 适用 | 通用 | 更专业：人物=价值观/决策，方法论=分析工具 |
| 上下文 | 不可控 | 用户决定，可预测 |

---

## 质量门禁

1. **路由是否准确**：涉及的轨别是否判断正确？
2. **用户选择是否被尊重**：是否按用户选的轨别+粒度执行？
3. **冲突是否处理**：人物间/方法论间/跨轨冲突是否被显式列出？
4. **双轨组合是否有价值**：两者是否真的互补而非重复？
5. **上下文是否可控**：是否只加载被选中的文件？
