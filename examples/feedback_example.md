# 完整反馈闭环示例

> 展示超级思考系统如何通过反馈循环持续优化

---

## 示例背景

**用户：** 小明（产品经理，工作3年）
**问题类型：** 职业发展
**当前时间：** 2025-05-10

---

## 第一次交互

### 1. 用户提问

```
用户：我在一家中型公司做产品经理，工作3年了。最近有一个机会可以跳槽到
一家创业公司做产品负责人，但需要放弃现在的稳定收入和title。我该怎么选择？
```

### 2. 系统处理

```

[创建默认画像] 生成基础配置

[路由推荐] 
  - 默认专家: 稻盛和夫(0.9), 德鲁克(0.9), 乔布斯(0.8)

```

### 3. 追问获取上下文

```
系统：
在你做决定之前，我想更好地理解你的情况：

1. 你现在的工作最看重什么？（稳定、成长、薪资、还是其他？）
2. 你对创业公司的了解有多少？有没有做过调研？
3. 你的经济储备能支撑多久没有稳定收入？
4. 如果创业失败，你的Plan B是什么？

请简单回答，这样我可以给出更贴合你情况的建议。
```

### 4. 用户回复上下文

```
用户：
1. 最看重成长和做成一件大事的机会
2. 了解一些，做过简单调研，但不确定
3. 大概6-12个月
4. 可以考虑回去继续做产品经理
```

### 5. 多专家分析

```
# 职业选择的多视角分析

## 问题
是否应该从现在稳定的产品经理岗位跳槽到创业公司做产品负责人？

---

## 视角分析

### 🌟 稻盛和夫视角：作为人，何谓正确？

**核心观点：**
- 做决定时，问问自己："这是作为人该做的吗？"
- 不是问"有没有好处"，而是问"应不应该"

**分析：**
从你的回答看，你看重"成长和做成大事"。这是好的发心。
但关键问题是：你是为了"逃避"现在的困境，还是"追求"新的可能？

- 如果是逃避 → 这不是正确的发心
- 如果是追求 → 问自己：有没有全力以赴地投入现在的工作？

**建议：**
在决定之前，先问自己一个问题：
"如果创业公司失败了，我会后悔这个决定吗？"
如果答案是"不会，因为我全力以赴尝试了"，那就去做。

---

### 📊 德鲁克视角：聚焦与放弃

**核心观点：**
- 战略的核心是"不做什么"
- 有效的管理者知道如何做选择

**分析：**
你提到了"放弃稳定收入和title"。这说明你在权衡取舍。

问题是：现在的公司真的没有成长空间了吗？还是你只是觉得"外面的更好"？

德鲁克会说：
"不要因为池塘小就羡慕大海，先问自己：你在现在的池塘里真的全力以赴了吗？"

**建议：**
尝试在现在的工作中找到一个"创业式项目"，
比如主动承担一个新产品的从0到1，这样既能测试自己，
又不用承担创业的风险。

---

### 🔥 乔布斯视角：Stay Hungry, Stay Foolish

**核心观点：**
- "Stay hungry, Stay foolish" - 保持渴望，保持愚蠢
- 连接过去和未来的点，才能找到人生的意义

**分析：**
乔布斯会说：你过去的经历（产品经验、职场历练）不是要放弃的东西，
而是要带到未来的"点"。

你现在的问题是：这些过去的点，能连接到创业公司这个选择吗？

如果能，这个选择就有意义。
如果不能，你可能只是在追逐潮流。

**建议：**
列出你过去3年积累的所有能力和资源，
看看创业公司这个机会能让你发挥哪些、扩展哪些。
如果80%以上的过去积累能用得上，那就勇敢去。

---

## 综合建议

### 你的情况分析

| 因素 | 现公司 | 创业公司 |
|-----|--------|---------|
| 成长空间 | 中等 | 高 |
| 风险 | 低 | 高 |
| 收入 | 稳定 | 不确定 |
| 你的准备度 | 高 | 中等 |

### 建议

**短期（1个月）：**
1. 深入了解创业公司：创始人背景、公司财务状况、产品市场匹配度
2. 和创始人深度聊一次，了解他对你的期望
3. 做一个"最小可行性验证"：比如用周末时间了解行业

**中期（3个月）：**
1. 如果验证结果积极，考虑骑驴找马找其他机会
2. 建立人脉网络，连接行业内的人
3. 储备6-12个月的生活费

**决策原则：**
如果满足以下3条，可以考虑跳：
- ✅ 你对创业公司的行业和方向真正感兴趣
- ✅ 你的过去积累能连接到未来
- ✅ 你有Plan B，且Plan B是你能接受的

如果只满足1条 → 建议不动
如果满足2条 → 深入调研后再决定
如果3条都满足 → 勇敢去

---

### 风险提示

⚠️ 创业公司失败率>90%，即使你有能力，也可能因为时机、市场、资金等问题失败
⚠️ 放弃稳定收入可能带来焦虑，影响判断
⚠️ 产品负责人意味着你要承担更多责任，不仅是产品，还包括团队、管理

### 下一步建议

1. 先不要辞职，用业余时间深入了解这家创业公司
2. 和创始人聊一次"如果我加入，你对我的期望是什么"
3. 列出你的"非卖品"（绝对不能放弃的东西），帮助做决策
```

## 用户画像演变

### 初始状态（新用户）

```yaml
expert_preferences: {}
question_patterns: 所有类型 count=0
routing_weights: 默认配置
```


## 关键洞察

### 1. 画像如何学习

| 反馈信号 | 画像更新 |
|---------|---------|
| 用户说某专家"有帮助" | 该专家 score +0.1 |
| 用户说某专家"没用/缺失" | 该专家 score -0.15 |
| 某类型问题评分高 | 该类型 preferred_experts 更新 |
| 某组合效果好 | 记录组合，下次推荐 |

### 2. 跨类型迁移

用户喜欢"乔布斯"在职业发展问题中的分析，
系统在"创业商业"问题中也会提高乔布斯的权重。

### 3. 组合追踪

当用户多次使用某组合且效果好时，
系统会在类似上下文下主动推荐该组合。

---

## 反馈循环的价值

### 没有反馈循环

- 每次分析都是独立的
- 专家选择依赖规则或随机
- 无法适应用户个人偏好

### 有反馈循环

- 系统记住用户的喜好
- 专家选择越来越精准
- 越用越好，真正成为"私人顾问"
