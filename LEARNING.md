# 超级思考自我优化系统

> 学习层 + 反馈层 = 持续进化的超级思考

---

## 核心架构

```
用户问题
    ↓
┌─────────────────────────────────────────┐
│  学习层（Learning Layer）                 │
│                                         │
│  1. 读取用户画像（Profile）             │
│  2. 结合画像路由推荐                    │
│  3. 追问获取上下文                     │
└─────────────────────────────────────────┘
    ↓
专家分析
    ↓
┌─────────────────────────────────────────┐
│  反馈层（Feedback Layer）                 │
│                                         │
│  1. 多维度反馈收集                      │
│  2. 写入选民画像                        │
│  3. 触发画像更新                        │
└─────────────────────────────────────────┘
    ↓
下次路由时读取更新后的画像
```

## 核心组件

### 1. ProfileManager（用户画像管理器）

**职责：**
- 管理用户画像的 CRUD 操作
- 提供基于画像的推荐
- 维护画像版本历史

**核心方法：**
```python
class ProfileManager:
    def get_profile(user_id) -> dict          # 获取用户画像
    def update_profile(user_id, feedback)     # 更新画像
    def get_recommendations(user_id, question) # 基于画像推荐
    def create_profile(user_id, name) -> dict # 创建新画像
```

### 2. FeedbackCollector（反馈收集器）

**职责：**
- 收集多维度反馈
- 结构化反馈数据
- 验证反馈完整性

**核心方法：**
```python
class FeedbackCollector:
    def collect(analysis_result) -> dict     # 收集反馈
    def validate(feedback) -> bool            # 验证反馈
    def extract_patterns(feedback) -> dict    # 提取模式
```

### 3. RoutingOptimizer（路由优化器）

**职责：**
- 基于反馈调整专家权重
- 追踪专家组合效果
- 推荐最佳专家组合

**核心方法：**
```python
class RoutingOptimizer:
    def adjust_weights(profile, question_type) -> dict  # 调整权重
    def track_combination(profile, experts, rating)       # 追踪组合
    def suggest_best_combination(profile, context) -> list # 推荐组合
```

### 4. ExpertCombinationTracker（专家组合追踪器）

**职责：**
- 记录专家组合的使用情况
- 计算组合成功率
- 发现最优组合模式

**核心方法：**
```python
class ExpertCombinationTracker:
    def record(experts, rating, context)      # 记录组合
    def get_success_rate(combination_key) -> float  # 成功率
    def find_best_for_context(context) -> list # 找最佳组合
```

## 学习流程详解

### 第一阶段：问题接收

```
用户输入问题 → 提取问题类型 → 加载用户画像
```

**问题类型分类：**
- `职业发展` - 职业选择、工作困境、职场晋升
- `人生规划` - 长期目标、生活意义、人生阶段
- `情感关系` - 亲密关系、家庭矛盾、友情
- `创业商业` - 商业模式、市场分析、融资
- `学术研究` - 论文选题、研究方法、学术方向
- `通用问题` - 无法归类的一般问题

### 第二阶段：画像增强路由

```
读取画像 → 分析偏好 → 调整专家权重 → 生成推荐
```

**路由决策矩阵：**

| 问题类型 | 默认权重策略 | 个性化调整 |
|---------|-------------|-----------|
| 职业发展 | 稻盛和夫+德鲁克+乔布斯 | +查理芒格思维模型 |
| 人生规划 | 尼采+加缪+苏格拉底 | +王阳明心学 |
| 情感关系 | 佛学+心理学视角 | +具体情况调整 |
| 创业商业 | 巴菲特+芒格+孙正义 | +行业专家视角 |
| 学术研究 | 费曼+西蒙+学科专家 | +跨学科视角 |

### 第三阶段：上下文追问

**追问模板（v5）：**

```
分析完成，请评价：

1. 总体评分（1-5）：___
2. 哪些视角最有帮助？（如：尼采的反抗精神、加缪的荒谬分析）
3. 哪些视角缺失或不够深入？
4. 结合你的背景，回复有帮助吗？（1-5）
5. 其他建议：

你的背景信息：
- 职业/身份：___
- 当前处境：___
- 核心困惑：___
```

### 第四阶段：分析执行

```
选定专家组合 → 并行/串行分析 → 整合输出
```

**分析模式：**
- **并行模式**：多个专家同时分析，适合多角度探索
- **串行模式**：专家依次深入，适合层层递进
- **混合模式**：先并行后串行，适合先广后深

### 第五阶段：反馈收集

```
展示分析结果 → 触发反馈收集 → 收集用户评价
```

**反馈维度：**
1. 总体满意度（1-5）
2. 有用专家（列表）
3. 无用/缺失专家（列表）
4. 上下文充足度（1-5）
5. 额外建议（文本）

### 第六阶段：画像更新

```
接收反馈 → 更新反馈历史 → 更新专家偏好 → 更新问题模式 → 调整路由权重
```

**更新规则：**
- 每次反馈追加到 `feedback_history`
- 专家被标记为"有用"→ 该专家 `success_count +1`
- 专家被标记为"缺失"→ 该专家 `success_count -1`（最低为0）
- 问题类型模式：相同类型问题计数 +1
- 组合效果：记录该组合的评分

## 冷启动策略

### 问题：用户画像为空时如何推荐？

**解决方案：**

1. **默认专家池**
   ```
   通用问题 → ["苏格拉底", "亚里士多德", "孔子", "佛学视角", "心理学视角"]
   ```

2. **基于问题的初始路由**
   - 问题包含"职业/工作"→ 添加管理学专家
   - 问题包含"创业/商业"→ 添加商业专家
   - 问题包含"人生/意义"→ 添加哲学专家

3. **快速建立画像**
   - 首次分析后立即收集反馈
   - 3次反馈后画像基本可用
   - 10次反馈后画像精准

## 性能优化

### 缓存策略
- 用户画像：实时读写，Redis 缓存
- 问题类型分类：LRU 缓存
- 专家组合推荐：基于反馈次数动态更新

### 批量处理
- 反馈收集支持批量更新
- 画像同步支持异步写入
