# 用户问题转专家问题 Prompt

## 任务

将用户的原始问题转换为专家视角的问题表述。

## 输入

**原始用户问题：**
```
{user_question}
```

**用户上下文：**
```
{context}
```

## 转换原则

### 1. 保留核心疑问
不要改变用户真正想问的问题，只转换问题的表述方式。

### 2. 增加专家视角
将开放式问题转换为专家可以具体回应的形式。

### 3. 明确问题类型

| 原始问法 | 问题类型 | 转换后 |
|----------|----------|--------|
| 怎么看/觉得... | 观点评估 | 从[领域]角度分析...的利弊 |
| 会不会/能否... | 可能性判断 | [领域]视角下，...的可能性及条件 |
| 为什么... | 因果分析 | [领域]认为...的根本原因是什么 |
| 怎么做/如何... | 方法求解 | [领域]对解决...有何方案和建议 |
| 哪个好/选择... | 比较决策 | [领域]对...进行多维度比较 |

## 输出格式

```markdown
## 问题转换

**原始问题**：{user_question}

**转换后问题**：

### 面向 {expert_name}（{domain}）
{converted_question}

### 面向 {expert_name2}（{domain2}）
{converted_question2}

### 面向 {expert_name3}（{domain3}）
{converted_question3}
```

## 示例

**原始问题**：AI会不会取代程序员？

**转换后**：

### 面向 技术乐观主义者（软件工程）
从技术发展趋势看，AI在哪些编程任务上已经/即将取代人类程序员？需要多长时间？

### 面向 怀疑论者（计算机科学）
AI在理解业务逻辑、创造性设计和复杂问题排查方面能否达到人类程序员的水平？

### 面向 人本主义者（社会学）
程序员被AI取代后，对从业者的职业发展和心理健康有何影响？如何应对？
