# Menu Extraction Prompt — 论点菜单提取

## 任务

从专家发言中提取核心论点，形成下一轮辩论的"论点菜单"。

## 四标准过滤规则

一个论点要被纳入菜单，必须满足以下全部标准：

| 标准 | 要求 | 排除 |
|------|------|------|
| **具体性** | 针对具体论点，不是泛泛而谈 | "X总体是好的"、"这个问题很复杂" |
| **可反驳性** | 有明确的判断，可以被支持或反驳 | "这个问题需要更多思考" |
| **论证性** | 有理由或证据支撑 | 只有观点没有理由 |
| **分歧性** | 与其他专家观点存在真实差异 | 和其他人说的一模一样 |

## 输入：专家发言列表

```
{{statements}}
```

## 论点编号规则

-格式：[A-R{n}-{序号}]，例如 A-R2-001 表示第2轮第1条论点
- 每轮论点独立编号
- 同一论点被多方引用时，编号跟随首次出现的位置

## 输出格式

```
## 第N轮论点菜单

### 核心论点（可反驳）
1. [A-RN-001] [专家A] 认为 [X]，因为 [Y]
2. [A-RN-002] [专家B] 认为 [P]，因为 [Q]
……

### 已收敛的论点（可选记录）
- [专家A] 和 [专家B] 都认为 [R]

### 建议下轮重点关注
- [专家C] 和 [专家D] 在 [S] 上分歧最深，可能需要进一步辩论
```

## 注意事项

- 只提取满足四标准的论点，无效表态直接过滤
- 论点必须包含"观点+理由"才算完整
- 相同论点合并，保留首次编号
- 过滤掉重复表述，保留核心差异
