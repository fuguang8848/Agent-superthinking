# 超级思考技能 (SuperThinking) v5

> 多视角深度分析 + 自我优化学习系统

---

## 核心概念

超级思考是一种多视角深度分析能力，通过调用不同领域的"专家"，为复杂问题提供多元思考角度。

**与普通分析的区别：**
- 普通分析：单一视角，可能有盲点
- 超级思考：多专家并行分析，视角互补，更全面

## 完整工作流 (v5)

### 流程图

```
用户问题
    ↓
┌─────────────────────────────────────────┐
│  学习层（Learning Layer）                 │
│                                         │
│  1. 读取用户画像（Profile）             │
│  2. 结合画像路由推荐                    │
│  3. 追问获取上下文                     │
└─────────────────────────────────────────┘
    ↓
专家分析
    ↓
┌─────────────────────────────────────────┐
│  反馈层（Feedback Layer）                 │
│                                         │
│  1. 多维度反馈收集                      │
│  2. 写入选民画像                        │
│  3. 触发画像更新                        │
└─────────────────────────────────────────┘
    ↓
下次路由时读取更新后的画像
```

### Step 1: 读取用户画像

**输入：** 用户ID、用户问题

**处理：**
```python
# 伪代码
profile = profile_manager.get_profile(user_id)
recommendations = routing_optimizer.get_recommendations(user_id, question)
```

**输出：**
- 用户的专家偏好
- 调整后的专家权重
- 该问题类型的推荐专家

### Step 2: 基于画像路由推荐

**决策矩阵：**

| 问题类型 | 默认专家 | 个性化调整 |
|---------|---------|-----------|
| 职业发展 | 稻盛和夫+德鲁克+乔布斯 | +查理芒格思维模型 |
| 人生规划 | 尼采+加缪+苏格拉底 | +王阳明心学 |
| 情感关系 | 佛学+心理学视角 | +具体情况调整 |
| 创业商业 | 巴菲特+芒格+孙正义 | +行业专家视角 |

**动态调整因子：**
- 用户历史评分高 → 该专家权重 +10%
- 用户近期使用 → 权重 +5%
- 用户标记为"缺失" → 权重 -15%
- 上下文匹配 → 权重 +20%

### Step 3: 追问获取上下文

**追问模板：**

```
在我开始分析之前，请提供一些背景信息：

1. 你的职业/身份是什么？
2. 当前面临的核心困境是什么？
3. 你已经尝试过什么方法？
4. 有什么约束条件需要考虑？

（可选）这个问题对你有多重要？
```

### Step 4: 多专家并行分析

**分析模式：**
- **并行模式**：多个专家同时分析 → 最后整合
- **串行模式**：专家依次深入 → 层层递进
- **混合模式**：先并行后串行 → 广度优先 + 深度探索

**输出格式：**

```markdown
# 多视角深度分析

## 问题
[用户问题]

## 分析

### 视角1: [专家名称]
**核心观点：**
- [观点1]
- [观点2]

**洞察：**
[深刻洞察]

---

### 视角2: [专家名称]
...
```

### Step 5: 整合与建议

```
## 综合建议

基于以上分析，我的建议是：

1. **短期行动**
   - [具体行动1]
   - [具体行动2]

2. **中期规划**
   - [规划1]

3. **长期视角**
   - [视角1]

## 风险提示
- [风险1]
- [风险2]

## 下一步
- [建议的下一步]
```

### Step 6: 多维度反馈收集

**反馈模板：**

```
分析完成，请评价：

1. 总体评分（1-5）：___
2. 哪些视角最有帮助？（如：尼采的反抗精神、加缪的荒谬分析）
3. 哪些视角缺失或不够深入？
4. 结合你的背景，回复有帮助吗？（1-5）
5. 其他建议：

你的背景信息：
- 职业/身份：___
- 当前处境：___
- 核心困惑：___
```

### Step 7: 更新用户画像

**更新内容：**
- 追加反馈到历史
- 更新专家偏好分数
- 更新问题类型模式
- 记录专家组合效果
- 重新计算路由权重

## 使用示例

### 示例1：职业困惑

**用户输入：**
> 我在一家大公司工作5年了，最近有机会跳槽到创业公司，但风险很大。我应该怎么选择？

**执行流程：**

1. **读取画像** → 发现用户偏好深度分析
2. **路由推荐** → 职业发展类，推荐：稻盛和夫、德鲁克、乔布斯、尼采
3. **追问** → 获取用户的具体情况和顾虑
4. **多专家分析** → 并行分析，输出多视角观点
5. **反馈收集** → 评价分析质量
6. **更新画像** → 记录本次交互

### 示例2：人生迷茫

**用户输入：**
> 我今年30岁，感觉人生没有意义，每天工作只是为了赚钱。我该怎么找到人生的方向？

**执行流程：**

1. **读取画像** → 新用户，使用默认配置
2. **路由推荐** → 人生规划类，推荐：尼采、加缪、苏格拉底、孔子、佛学
3. **追问** → 了解用户的具体处境和兴趣
4. **多专家分析** → 存在主义视角 + 东方哲学 + 心理学
5. **反馈收集** → 评价各视角的帮助程度
6. **更新画像** → 建立初步偏好

## 常见问题

### Q: 专家数量有限制吗？
A: 推荐3-5个专家，太多会影响整合质量。

### Q: 专家观点冲突怎么办？
A: 正常现象，不同专家有不同的假设前提。整合时点出分歧，让用户自己判断。

### Q: 用户不提供反馈怎么办？
A: 记录"未评价"，不影响使用，但画像更新会跳过该次。

### Q: 画像数据如何存储？
A: JSON文件本地存储，路径：`profiles/{user_id}.json`
